## 1. 产品概述

### 1.1 产品愿景

> **"让产品想法从 AI 对话到团队评审确认，一站式完成，不再有任何交接断层。"**

我们正在构建的是下一代 AI 原型协作平台——它不是另一个画图工具，而是原型交付这一高频协作场景的操作系统。从需求初稿到团队一致通过，链路中每一个角色的每一次沟通都发生在同一个上下文里。

### 1.2 产品定位

**一句话定位**：面向产品团队的 AI 原型交付与协作平台。

**差异化定位**：
- 传统原型工具（Axure/Figma）= 设计师和 PM **画** 原型的工具
- AI 原型工具（常见竞品）= AI **帮画** 原型的工具
- **我们的定位** = AI **生成 + 团队评审交付** 全链路协作平台

核心价值不在"AI 画图"，而在"消灭原型从生成到交付过程中的所有上下文切换"。

### 1.3 目标用户

| 用户角色 | 核心诉求 | 痛点 |
|---------|---------|------|
| **产品经理（核心用户）** | 快速把需求转化为可评审原型 | 原型与需求文档分离，评审时反复对照 |
| **团队负责人/总监** | 高效完成原型评审决策 | 评审版本混乱，链接传播不可控 |
| **UX 设计师** | 基于 AI 生成的骨架快速精调 | AI 生成结果难以直接复用为设计稿 |
| **开发工程师** | 获取精确的组件规格和需求标注 | 需求写在文档里，组件写在原型里，两边对不上 |
| **外部协作者（客户/供应商）** | 查看评审最新版本 | 拿到的是过期链接或多个版本搞不清哪个最新 |

### 1.4 核心价值主张

```
用户输入需求描述
    → AI 生成结构化原型（JSON + 组件实例，非图片）
    → 需求标注直挂页面组件（无需切换文档）
    → 版本迭代隔离管理
    → 唯一链接交付评审
    → 全程版本可追溯
```

## 2. 用户故事与使用场景

### 场景 1：PM 独自快速验证产品想法

**用户**：小雨（某 SaaS 公司 PM，刚接手一个 B2B 数据看板项目）
**背景**：凌晨 12 点，小雨刚和 CEO 头脑风暴完一个 CRM 新功能模块，她想在睡前把脑子里的想法快速变成可看的东西，明天早会直接演示。
**流程**：
1. 小雨打开工具，输入 prompt："设计一个 CRM 的客户详情页，包含基本信息区块、联系人列表、跟进记录时间线和快捷操作按钮"
2. AI 在 15 秒内生成 4 个页面的原型骨架
3. 小雨逐页浏览，对不满意的地方说"把联系人列表改成卡片形式"
4. AI 重新生成，小雨点击"发布预览"，复制链接发给 CEO
**结果**：早会上 CEO 直接在链接上标注了两个修改意见。

**验收标准**：
- [ ] AI 单次生成响应时间 ≤ 20 秒
- [ ] 生成后无需任何额外操作可直接预览
- [ ] 发布链接可在 10 秒内生成

### 场景 2：需求评审会——评审者和原型在同一个上下文

**用户**：老王（产品总监）+ 设计师小张 + 后端工程师阿强 + 外部客户李总

- [ ] 评审范围可按 R1/R2 筛选
- [ ] 外部用户无需注册即可评论（只读/评论模式）

## 3. 功能需求详述

### 模块一：需求阅读

#### 3.1.1 组件级需求标注

**功能描述**：在任意组件上添加需求标注（R1 必须项 / R2 期望项 / R3 参考项），标注内容与组件实例绑定，而非游离在文档中。

**交互说明**：
1. 鼠标悬停组件 → 右上角出现「+ 标注」图标
2. 点击图标 → 右侧滑出标注编辑面板
3. 选择标注类型（R1/R2/R3），输入内容
4. 支持富文本（加粗/列表/链接）
5. 按 Enter 保存，标注以 tag 形式浮现在组件右上角

**标注 Tag 展示规则**：
- R1：红色标签，优先级最高
- R2：橙色标签
- R3：灰色标签（默认折叠）
- 多个标注堆叠显示，点击展开列表

**验收标准**：
- [ ] 单个组件支持无限条标注
- [ ] 标注内容支持 2000 字符
- [ ] 支持标注的创建/编辑/删除/置顶操作
- [ ] 标注按类型筛选显示
- [ ] 标注变更记录（谁在何时修改了什么）

#### 3.1.2 标注与评审状态联动

**功能描述**：标注状态（待评审/已确认/有异议）直接在组件上反映颜色。

**状态机**：
```
待评审（灰色）→ 已确认（绿色）→ 有异议（黄色）
                        ↑__________|
```
**交互说明**：
- PM 点击标注状态按钮切换
- 有异议时必须填写原因
- 状态变更触发通知

**验收标准**：
- [ ] 状态变更记录时间线
- [ ] 状态变更通知（站内信/邮件）
- [ ] 支持按状态筛选组件

#### 3.1.3 评审模式

**功能描述**：进入评审模式后，隐藏编辑功能，全屏展示原型，评委可直接在组件上评论。

**交互说明**：
1. 点击「开始评审」→ 进入评审模式（URL 分享后外部用户自动进入此模式）
2. 点击任意组件 → 弹出评论输入框
3. 评论以聊天气泡形式浮现在组件旁边
4. 左侧显示评审进度：已评论/待评论组件数

**验收标准**：
- [ ] 外部用户无需登录即可评论（需手机号验证码）
- [ ] 评论支持回复和点赞
- [ ] 评审结束后可导出全部评论为 PDF/HTML

### 模块二：页面整理

#### 3.2.1 项目页面树

**功能描述**：左侧固定面板，以树状结构展示当前迭代的所有页面。

**信息维度**：
- 页面名称（可编辑）
- 页面状态（草稿/评审中/已确认/已交付）
- 入口关系（作为父级的子页面展示缩进）
- 评审范围（该页面涉及的 R1/R2 数量）
- 最近修改时间

**交互说明**：
1. 默认展示全部页面，支持搜索过滤
2. 点击页面名称 → 右侧画布加载该页面
3. 拖拽调整页面顺序和层级关系
4. 右键菜单：新建子页面/复制页面/删除页面/查看历史

**验收标准**：
- [ ] 支持最多 200 个页面的项目
- [ ] 页面树响应时间 < 200ms
- [ ] 拖拽操作有防误删确认

#### 3.2.2 页面状态管理

**状态定义**：
| 状态 | 颜色 | 含义 |
|------|------|------|
| 草稿 | 灰色 | 正在编辑，未进入评审 |
| 评审中 | 蓝色 | 已提交评审，等待确认 |
| 已确认 | 绿色 | 所有 R1 标注已确认 |
| 已交付 | 紫色 | 通过唯一链接发布给外部 |

**验收标准**：
- [ ] 状态变更自动记录到操作日志
- [ ] 状态变更触发页面树颜色更新
- [ ] 批量修改页面状态（多选 + 批量操作）

#### 3.2.3 页面关系图

**功能描述**：自动从页面结构生成入口关系图，帮助 PM 梳理产品结构。

**交互说明**：
1. 点击「页面关系图」按钮 → 全屏展示有向图
2. 节点 = 页面，边 = 跳转关系（由组件的链接属性推断）
3. 支持拖拽布局，高亮主路径
4. 支持缩放和平移

**验收标准**：
- [ ] 自动识别页面间的跳转关系
- [ ] 支持导出为 PNG/SVG

### 模块三：迭代版本管理

#### 3.3.1 迭代版本隔离

**功能描述**：每个项目支持创建多个迭代版本（如 v0.1/v0.2/v0.3），每个迭代拥有独立的页面列表和标注数据。

**数据结构**：
```
Project
└── Iteration (v0.1) —— 独立页面集合 + 独立标注
└── Iteration (v0.2) —— 独立页面集合 + 独立标注
└── Iteration (v0.3) —— 独立页面集合 + 独立标注
```

**交互说明**：
1. 点击「+ 新建迭代」 → 输入版本号（自动建议 v0.x）
2. 选择复制来源（可从任意已有迭代复制，或从空白开始）
3. 切换迭代时，画布完全切换到目标迭代的数据
4. 迭代之间页面可相互引用（生成参考链接）

**验收标准**：
- [ ] 支持最多 20 个迭代版本
- [ ] 迭代切换时间 < 500ms
- [ ] 支持版本对比视图

#### 3.3.2 版本快照与恢复

**功能描述**：每次"保存快照"创建不可变历史节点，可随时恢复到任意快照。

**快照元数据**：
- 快照名称（自动 + 可自定义，如"评审前版本"）
- 创建人 + 创建时间
- 包含的页面数 + 组件数
- 变更摘要（与上一快照的差异描述）

**验收标准**：
- [ ] 每次进入评审自动生成快照
- [ ] 快照恢复保留完整的组件数据和标注
- [ ] 快照数量无硬性上限（按存储计费）

#### 3.3.3 版本 Diff

**功能描述**：可视化对比两个版本之间的页面差异。

**Diff 展示**：
- 🟢 新增页面（绿色高亮）
- 🔵 修改页面（蓝色，展开显示变更组件）
- 🔴 删除页面（红色，带删除线）
- 组件级别的 diff：属性变化用黄色高亮

**验收标准**：
- [ ] 支持跨迭代选择两个版本对比
- [ ] 支持同迭代内两个快照对比
- [ ] Diff 报告可导出为 PDF

### 模块四：在线预览交付

#### 3.4.1 本地预览

**功能描述**：编辑完成后，点击「预览」直接在当前浏览器标签页模拟最终效果。

**预览能力**：
- 真实组件渲染（非静态图片）
- 基础交互模拟（按钮点击反馈、表单输入、页面跳转）
- 响应式预览（PC / 平板 / 手机三种视口）
- 标注显示/隐藏切换

**验收标准**：
- [ ] 预览模式覆盖 80% 组件的交互效果
- [ ] 视口切换无刷新
- [ ] 预览模式下禁止编辑

#### 3.4.2 云端发布

**功能描述**：一键发布到云端，生成永久可访问的唯一链接。

**发布配置**：
- 访问权限：公开 / 密码保护 / 仅协作者
- 水印设置：可选添加"预览版"水印
- 域名绑定：自定义子域名（企业版）
- SEO 信息：标题 + 描述 + 缩略图

**验收标准**：
- [ ] 发布链接生成时间 < 5 秒
- [ ] 链接格式：`https://{tenant}.aiprototype.app/p/{project_id}/{iteration_id}`
- [ ] 支持链接有效期设置

#### 3.4.3 多人实时预览

**功能描述**：同一链接被多人同时打开时，支持实时同步（一人滚动全员跟随）和独立浏览模式切换。

**模式说明**：
- **同步模式**：主讲人操作，其他人的视图同步跟随
- **独立模式**：每人独立浏览，可在自己的位置评论
- 模式切换按钮固定在底部工具栏

**验收标准**：
- [ ] 同步延迟 < 1 秒
- [ ] 支持最多 50 人同时在线（同一发布链接）
- [ ] 断线重连后自动恢复状态

## 4. 非功能性需求

### 4.1 性能

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 画布首次加载时间 | ≤ 3 秒 | 冷启动，从点击页面到可交互 |
| AI 生成响应时间 | ≤ 20 秒 | 从提交 prompt 到首屏组件渲染 |
| 操作响应延迟 | ≤ 100ms | 拖拽、缩放、标注等操作的体感延迟 |
| 大项目加载时间 | ≤ 5 秒 | 200 页面 + 1000 组件的项目 |
| 版本切换时间 | ≤ 500ms | 迭代版本间切换 |
| 预览发布生成时间 | ≤ 5 秒 | 从点击发布到链接可访问 |

### 4.2 可用性

- **SLA**：99.5% 月度可用性（计划内维护除外）
- **断线处理**：操作自动保存草稿，断线重连后同步
- **离线能力**：草稿编辑支持本地缓存（IndexedDB），重连后自动上传

### 4.3 安全

- 所有传输数据强制 HTTPS
- 密码保护链接使用 bcrypt 哈希存储
- 用户上传内容（图片等）自动扫描敏感信息
- 管理员后台日志保留 180 天
- 权限模型：Owner / Editor / Reviewer / Viewer 四级

### 4.4 兼容性

- **浏览器**：Chrome 120+ / Firefox 115+ / Safari 16+ / Edge 120+
- **移动端**：移动端查看器（响应式预览，不支持编辑）
- **操作系统**：Windows 10+ / macOS 12+ / Ubuntu 20.04+
- **屏幕分辨率**：最小支持 1280×720

### 4.5 合规

- 符合 GDPR（欧盟用户数据处理）
- 私有化部署版本（企业版），数据不出境
- 记录所有 AI 生成内容的用户操作审计日志

### 5.1 页面地图

```
登录 / 注册
```

## 6. 成功指标

### 6.1 北极星指标（North Star Metric）

> **"原型从生成到首次评审确认的平均时长"**

**定义**：用户从第一次点击「AI 生成」到该原型所有 R1 标注全部确认为「已确认」状态所经历的中位时间。

**目标值**：
- MVP 发布后 30 天：目标 ≤ 4 小时
- MVP 发布后 90 天：目标 ≤ 2 小时

**测量方式**：`事件埋点（prototype_created_at）→ 最后一个 R1 标注确认时间`

### 6.2 过程指标

| 指标 | 定义 | 目标值 |
|------|------|--------|
| **AI 生成采纳率** | AI 生成的页面中，被直接使用（含微调）的比例 | ≥ 60% |
| **标注覆盖率** | 有至少 1 条标注的组件 / 总组件数 | ≥ 40% |
| **评审参与率** | 收到评审链接的用户中，实际参与评论的比例 | ≥ 50% |
| **版本发布频率** | 团队每周平均发布评审版本次数 | ≥ 3 次/周 |
| **协作密度** | 每个活跃项目的平均协作者数 | ≥ 2.5 人 |
| **导出使用率** | 使用「导出开发文档」功能的用户比例 | ≥ 30% |
| **留存率** | 30 天后仍然活跃使用（≥ 3 次操作）的用户比例 | ≥ 40% |
| **核心用户 NPS** | 核心用户的净推荐值 | ≥ 40 |

### 6.3 指标采集方案

- **埋点平台**：自建事件采集 + 可选接入 Amplitude/Mixpanel
- **数据存储**：ClickHouse（时序数据）+ PostgreSQL（业务数据）
- **仪表盘**：内部 Grafana 仪表盘，核心指标每日更新

## 7. MVP 范围定义

### 7.1 MVP 原则

> MVP = 最小的有价值的原型交付闭环：**AI 生成 → 编辑 → 标注 → 评审 → 发布链接**

MVP 不追求功能完整，追求**让一个 PM 能够完整走完从想法到团队评审通过的全流程**。

### 7.2 优先级矩阵

```
         高影响
           ↑
    P1     │    P0
  (先行)   │  (MVP必做)
           │
低频率 ────┼─────→ 高频率
 (探索)    │   (核心场景)
           │
           │    P2
         (延期)
```

| 维度 | 竞品共同短板 | 我们的优势 |
